Most Common Features of a Car Dashboards:

  • Speedometer : This is the most common and important feature which is used by the driver since it represents the speed of the car.
  • tachometer : This is the also an important feature which is used by the driver since it represents the number of rotation per minute and this tell the performance of the engine.
  • Fuel Indicator : This is an important feature which is used to represent the fuel level and
  • Temperature Indicator : This is an important feature which is used to determine the engine temperature tells the condition of the engine to the driver.
  • Left/Right Signals : This feature is used when the driver has to take a turn Left/Right. This is important because it gives information to other drivers about the turn the driver making. It helps in reducing the accidents.
  • Hazard Lights : This feature is used when the driver has to Park and to notify other vehicles about the presence of the car. This represents that the car has some issues.
  • Door Indicator : This feature represents which door is open. This is also most used feature because it tells the specific door to close and it helps user to close that specific opend door easily.
  • Lights : This feature is used by the driver to know the Lights position. This feature allows driver to know the current lights positon and helps to change it or turn it off.
  • Check Engine Light : This feature represents the engine condition and tells that the engine has to be checked. This is also important feature because it represents that the engine has some issues.
  • Distance Covered : This feature is used by the driver to know the total distance covered by the car. This feature allows driver to reset the values so that a new record can be started.

Rare Uses of a Car Dashboards:

  • Park/Drive/Reverse Indicator : This represents the mode of the car. Since driver knows the mode he is in, this feature is very rarely used ny the driver.
  • Mileage Indicator : This represents the mode of the car. Since driver knows the mode he is in, this feature is very rarely used by the driver.
  • Steering Lock Indicator : This indicator turns on when the steering gets locked. This happens very rarly and with constant checkup this feature is almost never used by the driver.
  • Cruice Control Indicator : This indicator turns on when the car enters into cruice control mode. Since driver makes the decision of on/off cruice control it is also rarly used.
  • Extra Speedometer : Some interfaces has duel speedometer display (MPH & KMPH). Most of the drivers are used to their respective region units, making duel speedometer is rarly used by the driver.

The Controls That The User Interacts With The Car Dashboards:

Like mentioned in the "Most Common Features of Car Dashbaords", all these features interactes with the driver all the time.
  • The Speedometer, Left/Right Signals, Fuel Indicator are the most common features that interact with the driver when the driver is driving that car.
  • The Distance covered, Door Indicators, Engine Temperature are also important but not as frequently interacted to the user compared to the above mentioned features.
  • The Hazard Lights and Resetting the distance covered are the two features also directly interacted by the user. The User can turn on/off the hazard lights according to the need and the driver can reset the total distance covered at anytime.

Common Mistakes With The Current designs:

The current dashboard designs are prioritised accordign to the latest needs. Some missing the organization and some missing the staranderds and some are completly not relavent to the features. Like mentioned in the first section, those 5 dashboards have their own pros and cons but not one of the design has better way of helping driver in all possible ways.
  • If the organization is perfect the speedometer and tachometer is not adjusted to the user convenient. It is designed to make the dashboard pretty.
  • If the speedometer, tachometer, fuel indicator are well organised the rest of the indicatiors are not particullarly designed to help the user to track those signals but to coverup the space in the dashboard.
  • The Color representation is also not properly utilised because of using wrong color at wrong time will defenity confuse the user at all time.
  • Making a congested dashboard might provide space for other requirments but it will not help the driver at all. The representation should be well organized and should be easyly understandable by the user.

Improvements To make On The Control Interface:

By considering all the pros and cons from the mentioned 5 dashboards, these are the best suggestions which makes the dashboard well organised and easily understandable by the driver.
  • Providing a place for all indicators in an order (alphabetically or commonly used indicators) makes the user to easily tack the data.
  • By having larger speedometer and smaller tachometer helps user to track the speed without any delays and makes user comfortable in getting the right information at right time.
  • By adding specific colors to the specific features also helps user to understand the importance of the indicators.
  • The colors also help in tracking the importance of all the features so implementing the colors in Speedometer, tachometer and fuel indicator will help the driver in getting the information faster and helps the driver.
  • Insted of showing the door indicator that "Doors are open", showing which door is opend will help driver to close that specific door. This helps driver in saving time insted of checking all the doors.
  • Eventhough the Park/Drive/Reverse indicator is rarly used by the driver, making it with a large font is essential because it represents the mode the car.

Initially the car will be in Park mode and to move the car we have to change the mode to either Drive or Reverse. To change mode press "D" for Drive, "R" for reverse, "P" for parking.
Park Mode
Drive Mode

Once the Mode is changed, the user can press "Up arrow" key which is accelaration. The design has digital number representation and color representation aswell. When the speed is between o-59, it shows green color. When the speed is between 60-119, it shows yellow. When the speed is between 120-180, it shows red color.

Similarly the Tachometer also changes color at intervels 3 and 6 i.e, 0-3 RPM will give green color, 3-6 it gives Yellow and 6-9 gives red color.

Simultaniously the Fuel indicator also decreases the value since the vehical is moving. The color representation is implemented as if the fuel is full then it shows green color. As fuel is decreased by 30% then the color is changed to yellow and if it is at last 30% it is represented with red color.

All these are shown in the below screenshots.
The "Left Arrow" key and "Right Arrow" key are used to take left and right turns respectively. The left blinker blinks on pressing the "Left Arrow" key and the right blinker blinks on pressing the "Right Arrow" key.
Left Signal
Right Signal
The Cruice Control is also implemented in this design. By pressing "c" key the cruice control actives. At the bottom the interface the Cruice Control indicator is enabled when the cruice control is activated. By pressing "c" again will disable the cruice control.
The Indicators on either side of the screen so that the driver can easily track the indicators and it doesn't take much time to understand the indicator.
I have implemented the checking page to check the indicators working status. All the indicators are validated through this page. To refill the fuel or the buttons for hazard lights and to reset the total distance are implemented in this page.
By clicking each button that represented button gets activated which represents that specific indicator has issues and it blincks with red color. By clicking it again, the interface takes it as the issue is resolved and the indicator will go back to normal.
The Lights are also controlled from the checking page. once the Lights button is clicked the lights are turned to Dipped Beam and if it is again clicked the lights are turned into Full Beam. If th button is again clicked the lights will get turned off.
The Door indicators are shown seperatly to each door. The bonnet and trunk are also have indicators when they are open. The priority is given to the bonnet and then to the trunk and then to the normal doors.

BONNET -> TRUNK -> DOORS
This design has hazard light implementation also. By clicking the hazard button from checking page the both Left and Right indicators stats blinking. This hazard light active indicator is showed at the bottom of the interface beside cruice control indicator.
The total distance can be reset using the reset button. This button is available in checking page and by clicing it the total dstance will be reseted to 0.
Once the fuel is completed the engine automatically stops by gradually decreasing the speed and makes Engine Start button active. Without refilling the fuel the Start Engine button will not work.
Auto Decrease the speed when fuel is empty.
When fuel is empty, engine doesn't start.
To Refill the fuel, the Fuel Refill button is added to the checking page. By clicking and holding on to the Fuel Refill button the fuel will be filled and it can be seen in the interface.
Finally the temperate is displayed on the bottom of the screen. The temperature is measurred in Fahrenheit.
